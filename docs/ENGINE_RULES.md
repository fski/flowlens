## 3. Tab Walk Event Types

Generated by `tabWalk()` in `a11y-audit-snippet.js`. Blocking types are defined in `TAB_BLOCKING_TYPES` (`panel.js:122-129`).

| Type | Blocking? | Description |
|------|-----------|-------------|
| `possible_focus_trap` | Yes | Element appears multiple times in tab order — likely a focus loop |
| `non_dialog_focus_trap` | Yes | Focus loop outside a dialog — container traps focus |
| `roach_motel` | Yes | Focus enters but cannot leave |
| `dialog_focus_not_trapped` | Yes | Modal dialog is open but focus escapes to sibling content |
| `focus_on_body` | Yes | Focus returned to `<body>` — likely a loader chain issue |
| `focus_failed` | Yes | Element did not accept focus as expected |
| `focus_jump` | No | Focus jumped across distant subtrees |
| `focus_thrashing` | No | Many focus changes in short time — loader mount/unmount churn |
| `duplicate_in_order` | No | Element appears multiple times in tab order |
| `role_interactive_not_focusable` | No | `role=button/link` but element is not focusable |
| `dialog_no_focusables` | No | Open dialog with no focusable elements inside |

## 4. How to Add a New Rule

### Step 1: Register in RULE_REGISTRY

Add an entry in `a11y-audit-snippet.js:224-282`:

```javascript
MY_NEW_RULE: {
  id: "MY_NEW_RULE",
  wcag: "X.X.X",
  level: "A",           // "A" | "AA" | "AAA"
  confidence: "strict",  // "strict" | "heuristic" | "advisory"
  run: null,            // overridden if the rule has a dedicated function
},
```

### Step 2: Implement the logic

Add the check in the `run()` block in `a11y-audit-snippet.js` (after line ~870). Use `add(findings, {...})`:

```javascript
add(findings, {
  type: "MY_NEW_RULE",
  el: targetElement,
  severity: "medium",
  note: "Description of the problem.",
  extra: { /* evidence fields */ },
  fix: "Suggested fix.",
});
```

### Step 3: Add a fix suggestion

Add to `FIX_SUGGESTIONS` (`a11y-audit-snippet.js:160`):

```javascript
MY_NEW_RULE: 'Description of how to fix this problem.',
```

### Step 4: Add evidence safely

Fields in the `extra` object must be:
- Serializable (no DOM references, no circular structures, no functions)
- Bounded (truncate long strings, avoid large arrays)

### Step 5: Add a fixture

In `fixtures/a11y-rule-fixtures.html`:
- Add element(s) with expected flagging.
- Add element(s) with expected non-flagging (negative case).
- Add the expected count to `docs/A11Y_RULE_FP_AUDIT.md`, section 4.

### Step 6: Verify

`node --check a11y-audit-snippet.js` must pass.

## 5. FP Hotspots

For the full false-positive audit and precision plan, see [A11Y_RULE_FP_AUDIT.md](./A11Y_RULE_FP_AUDIT.md). Summary of known hotspots:

| Rank | Rule | FP reason | Mitigation |
|------|------|-----------|------------|
| 1 | `FOCUS_VISIBLE_SUPPRESSED` | At-rest computed style only; `:focus-visible` styles in cross-origin stylesheets are invisible | Scans stylesheet rules; downgrades to advisory when unresolvable |
| 2 | `CLICK_WITHOUT_KEYBOARD` | Ancestor/global key handlers cannot be proven | Ancestor delegation treated as unproven (advisory, never strict) |
| 3 | `ARIA_HIDDEN_FOCUSABLE` | Focus guard sentinels and inert patterns | Exempts `data-focus-guard`, sentinel signatures, 1×1 guards; transition ticks downgraded to advisory |
| 4 | `TOUCH_TARGET_TOO_SMALL` | Cannot detect wrapper/pseudo hit areas | Confidence downgraded to heuristic; inline text links exempted |
| 5 | `IFRAME_MISSING_TITLE` | Presentational/AT-ignored iframes | Exempts `aria-hidden="true"` and `role="presentation\|none"` |
| 6 | `LOADER_WITHOUT_ANNOUNCEMENT_HOOK` | DOM-presence heuristic only; cannot prove runtime announcements | Kept heuristic |
| 7 | `DUPLICATE_MAIN_LANDMARK` | Microfrontend composition with isolated mount islands | No mitigation — expected in MFE contexts |
| 8 | `HEADING_HIERARCHY_FRAGMENTED` | Stitched MFEs with independent heading roots | No mitigation — expected in MFE contexts |

2. Paste `a11y-audit-snippet.js` into the DevTools console.
3. Run `A11YFlowAudit.run({ strict: true })`.

   const byType = A11YFlowAudit.last.findings.reduce((m, f) => {
